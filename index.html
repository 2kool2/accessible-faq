How it works

Takes a dl list and wraps each of its dt content with a button. The dt is targeted by the data-pab attribute which has the value of the id of the dd to show or hide.

Adding the data attribute data-pab-expand to a dt will expand a dd on page-load.

If a URL fragment identifier matches the id of a FAQ dd then it is expanded on page-load.

The code is very semantically written with one caveat. Only a single dd per dt is allowed, which is fine for an FAQ, but not for a dictionary which would require multiple responses.
